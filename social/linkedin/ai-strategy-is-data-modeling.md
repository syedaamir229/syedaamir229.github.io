# LinkedIn companion: Most AI Strategy Is a Data-Modeling Problem in Disguise

Most AI strategy is a data-modeling problem wearing a costume.

The costume has a demo, a vendor, and a budget line with "AI" in it. Underneath, what decides whether any of it ships is almost never the model. It's whether the business has one governed answer to "what is a customer, what is churn, what is an active subscriber," living somewhere every system can read.

I've watched two AI projects off the same stack end in opposite places. One inference model shipped in about four months, because the features it needed already lived in a governed table built years earlier for dashboards. One natural-language agent demoed like magic, then handed an executive a confidently wrong number, because "active subscriber" was never governed and it reached for one of four definitions.

Same lesson both times: the model was interchangeable, the data model was destiny.

Here's the cheap test for whether your AI strategy is real. If you swapped the model for a different vendor's next year, would the plan survive untouched? If yes, it was never about the model, and your data team already knows it. If the plan collapses, the model was load-bearing, and outside actual research that is rare.

Most of us aren't doing research. We're trying to get a reliable answer into a system that acts on it. The frontier there isn't the model. It's the definitions.
